studiojoe-viz · showcase

Five techniques. Five self-contained demos.

Each page is a standalone HTML file — no build step, no server. Open one and it runs. The audio is faked (sine-based pulses) so you can see the reactivity without wiring up Spotify or a mic. These are the patterns documented in references/patterns.md, pbr.md, and performance.md, applied to one-off prototypes.

01

Fluid Ink

Bilateral metaball SDF with dual-time drift. Monotonic time drives the jagged edge noise; an oscillating node-time sloshes the ink sideways like a tilting canvas.

SDFmetaballsdual-time drift
02

Chrome Orb

Physically-based icosahedron with metalness 1.0 + ACES tone mapping + UnrealBloom. Procedural PMREM environment map supplies the reflected stage lights.

PBRenvMapbloom
03

Kaleidoscope Tunnel

Single full-screen fragment shader. 6-fold polar fold with mid-driven twist, tunnel depth accumulator, HSL palette rotation, ring + spoke gem flares.

fragmentpolar foldHSL
04

Particle Waves

Canvas 2D. 400 particles on sinusoidal paths with persistent trails. Beat onsets trigger a radial expansion and color flip; between beats, the field drifts.

Canvas 2Dparticlesbeat-reactive
05

Wire Terrain

Three.js PlaneGeometry with vertex-shader displacement. Three-octave noise + bass-driven amplitude. Wireframe material + slowly orbiting camera.

vertex shaderdisplacementwireframe

All demos use three@0.128 via CDN. The fake audio frame mirrors window.AudioEngine.currentFrame() so code transplants into the registry cleanly. Each file is a valid starting point for a musicplayer-viz/prototypes/ entry. See the skill's new-viz-recipe.md for the full Phase 1 → Phase 4 integration path.
